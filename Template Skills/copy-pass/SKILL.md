# Copy Pass

## Purpose

Make UI language clear, honest, calm, and consistent with product truth.

## Workflow

1. Read product requirements, architecture constraints, and current UI states.
2. Identify the user’s question in each state:
   - What is happening?
   - What can I do?
   - What will happen if I continue?
   - What did not happen?
3. Write short copy by state.
4. Preserve backend error boundaries and confirmation semantics.
5. Flag any copy that implies unsupported product behavior.

## Rules

- Do not promise unsupported capabilities.
- Do not make AI sound more certain than its inputs allow.
- Do not hide destructive consequences.
- Do not rename canonical product concepts casually.
- Prefer direct, operational language over marketing flourish.

## Output

1. Task
2. Stage
3. Copy problem
4. Recommended copy
5. State coverage
6. Product truth preserved
7. Blockers
