# Workflows

Load a workflow only when its task matches. Role files define authority; workflows define repeatable
process. Keep project-specific commands, schemas, and test data in the target repository, not here.

| Workflow | Use for |
|---|---|
| \`architecture-audit\` | architecture review, duplication, cleanup, prioritization |
| \`backend-contract\` | API, server action, persistence, integration, schema slices |
| \`backlog-intake\` | feedback, screenshots, bugs, and improvement capture |
| \`frontend-design-system\` | UI, design-system, responsive, and interaction work |
| \`plan-writing-closeout\` | active plans, checkpoints, archive, and source-of-truth reconciliation |
| \`prompt-handoff\` | Product-owned routing and exact next-role prompts |
| \`qa-browser-regression\` | browser, responsive, persistence/readback, and cleanup proof |
| \`domain-doctrine-audit\` | domain quality, safety, and doctrine review |
