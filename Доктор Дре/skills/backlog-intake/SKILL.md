# Backlog Intake

## Purpose

Capture a durable, implementation-neutral work item without losing the user's evidence or creating a
duplicate task.

## Workflow

1. Preserve the observed symptom, route, target, screenshot or other evidence, and desired outcome.
2. Search active work and accepted decisions for an existing owner.
3. Classify the item: bug, improvement, question, decision, or deferred idea.
4. Name the likely canonical owner and the minimum acceptance evidence.
5. Record only durable facts; do not invent source causes before inspection.

## Rules

- Do not implement or route a speculative solution as if it were established.
- Do not create duplicate work for an already active task.
- Keep the item concise; link evidence instead of duplicating it.

## Output

Title, Priority, Symptom, Evidence, Desired outcome, Likely owner, Scope, Acceptance boundary,
Non-goals, Open questions.
