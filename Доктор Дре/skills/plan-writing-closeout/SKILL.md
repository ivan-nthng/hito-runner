# Plan Writing And Closeout

## Purpose

Keep plans useful as source-of-truth work boundaries rather than long-running activity logs.

## Workflow

1. Read the active plan, latest implementation/QA reports, and current-state documents.
2. Separate accepted work, active work, blockers, and deferred follow-up.
3. Update only the durable facts that affect future ownership or product truth.
4. Close or archive a plan only when its exit criteria and required evidence are actually satisfied.
5. Reconcile indexes or dashboards if the repository uses them.

## Rules

- Documentation follows implementation; it does not claim unbuilt behavior.
- Do not create a new plan for a trivial correction that fits an existing task.
- Prefer concise decisions and links to evidence over pasted command logs or agent transcripts.
- Preserve historical records; do not rewrite accepted history to make the present look simpler.

## Output

Plan status, Evidence reconciled, Current truth updated, Deferred work, Next owner, Blockers.
